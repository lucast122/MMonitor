Introduction

MMonitor is software designed for monitoring metagenomes over time, using data from nanopore sequencing. It includes a Python application for desktop use, which manages the analysis of sequence data and metadata for taxonomic and functional insight, as well as quality control. The accompanying webserver Dashboard provides a platform for users to view their analysis results after logging in. Developed with input from laboratory researchers, MMonitor supports real-time data analysis to offer immediate sample overviews, and is flexible for use on personal computers or via external servers for enhanced processing capabilities.

Installation and Usage Instructions

MMonitor is a comprehensive tool comprising two main components: a Desktop Application and a Webserver for visualization. The Desktop Application, designed for pipeline execution, should ideally be installed on the computer managing the nanopore sequencer or one that has access to the sequencing data. It offers the flexibility of being operated through a user-friendly graphical interface or via a command line tool on a remote server. Although MMonitor was designed for handling metagenomes, it could also be used to inspect isolates, e.g. checking them for contamination.

    MacOS

    1. Download the prebuilt MMonitor application from this website.
    2. There is no installation necessary just unzip the application and open the file 'mmonitor'.
    3. Wait for the application to start. If the application doesn't start check the console output for an error message. Proceed with general usage instructions below.

    Unix

    Currently, if you want to run MMonitor on a Unix distribution you will need to clone thegithub repositoryand run the code directly using Python.
    1. Open a command line tool like Terminal (Unix) or PowerShell (Windows). Make sure you have Git installed on your system and run:

      git clone "https://github.com/lucast122/MMonitor/"
    2. Optional: Create a new Conda environment to avoid conflicting versions:

      conda create -n mmonitor python=3.10

      conda activate mmonitor
    3. Install the required Python dependencies with:

      python -m pip install -r MMonitor/desktop/requirements.txt
    4. Finally, run the software using:

      python MMonitor/desktop/src/mmonitor/__main__.py

    Windows

    Warning: Running analysis pipelines is not possible using MMonitor's Desktop application with windows as the tools used in the pipelines were not designed to work with Windows. The only thing you can do with MMonitor on Windows right now is adding metadata to a sample and opening local MMonitor databases for result inspection. You can still inspect your results on this website with Windows, however, adding new data is only supported under MacOS or Unix.

    Currently, if you want to run MMonitor on Windows you will need to clone thegithub repositoryand run the code directly using Python.
    1. Open PowerShell (you can search for it in the Start menu). Ensure you have Git installed on your system. If Git is not installed, you can download and install it from here. Then run the following command to clone the repository:

      git clone "https://github.com/lucast122/MMonitor/"
    2. Optional: It's recommended to create a new Conda environment to avoid conflicting versions with other Python projects. If Conda is not installed, you can download and install Anaconda from here. To create and activate a new environment, run:

      conda create -n mmonitor python=3.10

      conda activate mmonitor
    3. Install the required Python dependencies. Navigate to the directory where you cloned the repository and run:

      python -m pip install -r MMonitor\desktop\requirements.txt

       Note: Use backslashes (\) for file paths in Windows.
    4. Finally, to run the software, use the following command:

      python MMonitor\desktop\src\mmonitor\__main__.py

    General usage instructions

    1. Login or register an account on this website by clicking either Login or Register on this website.
    2. After installing MMonitor run the application and a window similar to the one below should pop up.
    3. Click on User Authentication. The 'User Authentication' window should pop up. The credentials for a test user should be displayed. Do not change the host. Change the user and password to the username and password you chose when registering for this website. Make sure everything is correct and click'Save config' to save the changes. The user that is provided will be associated with all the data that is evaluated with MMonitor.
    4. Alternatively you can also skip step 5 and instead use a local database for storing your data. To do this click on 'Create local DB' to create one, or 'Choose Local DB' to choose a DB created earlier. If you choose or create a DB your data will automatically be inserted in that DB when processing sequencing files. After adding data click 'Start dashboard' to start a local server with your data.
    5. Add your metagenome data. Click 'Process sequencing files'. In the new window select any analysis that you wish to perform. For taxonomic analysis select 'Quick taxonomy nanopore' if you are using nanopore whole genome sequencing reads. If you only sequenced the 16s rRNA gene select 'Quick taxonomy nanopore 16s'. For functional analysis select all the steps you wish to perform, however, when selecting e.g. 'KEGG' you will also need to select the previous steps as they are required. In example you need an assembly for the KEGG pathways and annotation, but you can run only the assembly without running the other functional anlysis steps.
    6. After selecting the analysis hit submit and the Input Window should open. You can either add a single sample, by filling out all necessary fields (Sample Name, Project Name, Sample Date, ...) and then clicking 'Add FASTQ from folder'. Select the folder with the sequencing data (when using Minknow for sequencing this should be called fastq_pass). After selecting the folder all sequencing files that will be considered for the analysis will be displayed. If you want to add multiple you can do so by providing a CSV file that contains the information you would otherwise fill out in the window as well as the folder path to the sequencing files. Every column in the CSV file will be treated as an extra sample. If you have different barcodes select 'Multiplexing' before adding a CSV and provide the barcode IDs in the respective column of the CSV file. If 'Multiplexing' is not selected, then the barcode column will be ignored. An example CSV file can be downloaded here. Finally, click 'Submit' to start the analysis either for one sample or multiple samples in case you provided a CSV.
    7. When the analysis is finished a popup will notify you. Go to this website and click 'Dashboard' to see the results. If you used a Local Database instead then click 'Start Dashboard' in the main window to see your results instead.

Requirements

MMonitor supports MacOS, Ubuntu, and Windows. Currently there is only a build for MacOS on the website, if you want to use MMonitor on a different OS please refer to the github page. The prerequisites include a modern CPU, 16-32GB of RAM, and an up to date browser like Safari or Google Chrome.
